Hardware & Firmware

Advanced embedded system design with optimized power consumption and medical-grade sensor accuracy

The AURA Smart Health Watch represents a sophisticated embedded system designed for continuous health monitoring and emergency response. Built around the ESP32-C3 Super-mini microcontroller, the device integrates multiple sensors including the MAX30102 for heart rate and SpO‚ÇÇ monitoring, MPU6050 for motion tracking and fall detection, and a GPS module for real-time location tracking. The compact PCB design houses all components in a wearable form factor while maintaining medical-grade accuracy and reliability.

Power efficiency is achieved through intelligent firmware that leverages ESP32 deep sleep modes, event-triggered GPS activation, and optimized sensor polling rates, extending battery life to 24+ hours of continuous operation. The watch communicates seamlessly via Bluetooth Low Energy (BLE) with the mobile application, transmitting vital signs data every 15 minutes while maintaining the capability for immediate emergency alerts through the SOS button and automatic fall detection algorithms. A TFT LCD display provides real-time visualization of health metrics, ensuring users stay informed about their wellness status throughout the day.

System Architecture

A comprehensive microservices-based architecture connecting wearable hardware, intelligent analytics, and user interfaces for seamless health monitoring

Data Flow Overview

The Smart Watch captures raw physiological data through onboard sensors (ESP32) and transmits it via MQTT/WebSocket to the Main Backend (Node.js/Server). The backend forwards telemetry streams to the Diagnostic Core (Python FastAPI) for real-time analysis using AI models. Critical alerts are pushed back to the User Dashboard (Web/Mobile App) instantly, while context and summaries are enriched through Google Gemini GenAI API for professional medical reporting.

Model-View-Controller (MVC) Architecture

SCALABLE & MAINTAINABLE DESIGN

The backend follows a clean MVC architectural pattern that separates concerns between data models, business logic, and request handling, ensuring code maintainability and scalability.

  • Model Layer: MongoDB schemas for Users, Patients, Doctors, Devices, Alerts, and Reports with data validation
  • Controller Layer: Handles HTTP requests, validates inputs, enforces RBAC, and executes core workflows
  • View Layer: RESTful API responses consumed by frontend platforms (mobile app and web dashboard)
  • Real-time Layer: WebSocket connections for live health data streaming and instant alerts
üîå

RESTful API Architecture

STANDARDIZED COMMUNICATION

Comprehensive REST API design following industry best practices with resource-oriented endpoints, standard HTTP methods, and consistent response formats.

API Categories:

  • Authentication & User Management: Registration, login, email verification via OTP, password reset, social auth (Google/Facebook)
  • Patient Management: Profile updates, doctor browsing, medical reports access, device pairing
  • Doctor Management: License verification, patient monitoring, medical notes, profile management
  • Admin Management: User management, doctor approvals, system monitoring, platform control
  • Device Management: Wearable inventory, device-patient pairing, firmware updates

Design Principles:

  • Stateless Communication: Each request contains all required information including JWT tokens
  • Standard HTTP Methods: GET, POST, PUT, DELETE mapped to CRUD operations
  • Consistent Status Codes: Proper use of 200, 201, 400, 401, 403, 404, 500 responses
  • Input Validation: All requests validated before processing to ensure data integrity

MongoDB Database Design

NOSQL FLEXIBILITY & SCALABILITY

Document-based NoSQL database architecture using MongoDB Atlas for cloud hosting, providing flexibility, horizontal scaling, and high-performance data operations.

Key Collections:

  • Users: General information, email, password (bcrypt hashed), role, authentication details
  • Patients: Health profiles, medical history, paired devices, vital signs records
  • Doctors: License numbers, specializations, patient lists, medical notes
  • Devices: Available wearables, device specifications, pairing status
  • Alerts: Emergency notifications, timestamps, severity levels
  • Reports: AI-generated medical summaries, health trends, analytics

Why MongoDB:

  • Flexible Schema: Accommodates evolving user profiles without complex migrations
  • Horizontal Scaling: Sharding support for handling growing data volumes
  • Fast Operations: High-performance reads/writes for real-time health monitoring
  • Cloud Integration: MongoDB Atlas provides automated backups and high availability

Authentication & Authorization

JWT & ROLE-BASED ACCESS CONTROL

Multi-layered security system protecting sensitive healthcare data through JWT-based authentication and granular role-based access control (RBAC).

Authentication Features:

  • Password Security: Bcrypt hashing with salt rounds for secure password storage
  • JWT Tokens: Signed tokens with expiration times to prevent session hijacking
  • Email Verification: OTP-based email confirmation with retry limits and expiration
  • Social Authentication: OAuth integration with Google and Facebook providers
  • Password Reset: Secure token-based workflow for forgotten passwords

RBAC System:

  • Patient Role: View/update own profile, access reports, browse doctors
  • Doctor Role: Manage profile, view patient data, leave medical notes, license verification
  • Admin Role: Full system control, user management, doctor approvals (manual creation only)
  • Middleware Enforcement: Every API endpoint validates JWT and enforces role restrictions

Real-time Data Processing

WEBSOCKET & DATA PIPELINES

Advanced data pipeline architecture handling real-time sensor data from wearables, immediate storage, AI processing coordination, and instant alert distribution.

Data Flow Lifecycle:

  • 1. Data Ingestion: Wearables send health metrics via WebSocket connections every 15 minutes
  • 2. Immediate Storage: Raw data validated and stored in MongoDB for historical tracking
  • 3. DS Server Forwarding: Data forwarded to Python FastAPI Data Science server for AI analysis
  • 4. Report Reception: Receives processed reports, predictions, and critical alerts from DS server
  • 5. Alert Distribution: Pushes critical alerts to patient mobile app and doctor dashboard in real-time

Performance Optimization:

  • Connection Pooling: Efficient MongoDB connections for handling concurrent requests
  • Indexed Queries: Database indexes on userId, deviceId, timestamps for fast lookups
  • Async Processing: Non-blocking operations for email, notifications, and data forwarding
  • Horizontal Scaling: Load balancer support for distributing API traffic across multiple instances

Isolation Forest AI Model

UNSUPERVISED ANOMALY DETECTION

Advanced machine learning model that detects health anomalies without requiring extensive labeled medical datasets. The model learns "normal" health patterns and flags deviations automatically.

Hybrid Alert Logic System

ACTIVE SAFETY MONITORING

Three-tier alert system combining hard safety thresholds, AI analysis, and time-window validation to eliminate false alarms while ensuring no critical events are missed.

Alert Levels:

  • RED Alert (Critical): Heart Rate > 120 BPM or SpO‚ÇÇ < 90% triggers immediate notification
  • YELLOW Alert (Warning): AI-detected anomalies that don't meet critical thresholds
  • GREEN Status (Safe): All vitals within normal parameters and no anomalies detected

Dynamic Time-Windowing:

Filters out motion artifacts and sensor noise by validating readings over a short time period instead of reacting to single data points, significantly reducing false alarms.
